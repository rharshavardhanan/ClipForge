# AVSS v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the self-improving layer — audience simulator, 3-variant bandit testing, consistency regulator, edit-DNA elite templates, epsilon-greedy editing policy, and an RL loop from real YouTube stats — to the main clip pipeline.

**Architecture:** A new pure-logic `src/avss/` package introduces an explicit **EditPlan** (every render decision) and **SourceSignals** (per-clip word/RMS/semantic evidence). Variants of the plan are generated, clamped by the regulator, scored by the deterministic plan-level simulator, and only the winner is rendered. `clipforge stats` pulls real YouTube metrics, computes the spec reward, updates the policy arms, and promotes ≥70 %-retention edit DNA to `./elite_templates/`. Design doc: `docs/superpowers/specs/2026-07-03-avss-v1-design.md`.

**Tech Stack:** TypeScript (ESM, Node 20), vitest, Remotion (props-driven), YouTube Data API v3 + YouTube Analytics API v2 via plain fetch.

## Global Constraints

- Semantic subscores from the LLM layer are **0–10**; `SourceSignals.semantic` is normalized to **0–1**.
- Explore ONLY: hook text source, caption preset (within the mode's family), zoom timing/intensity, SFX volume/on. **Framing is never a variant dimension.** One render per clip, always.
- Explicit user flags pin dimensions: `--style`/`caption` pins captionPreset, `--no-zooms` pins zooms off, `--no-sfx` pins sfx off.
- AVSS is fail-soft: any error inside variant/simulate falls back to the regulated base plan; a clip export never fails because of AVSS.
- All new core logic is pure (I/O at edges), mirrored under `tests/avss/`, imports `../../src/avss/<mod>.js`.
- Simulation tick = 0.5 s. Reward weights (spec): 0.35 retention, 0.20 completion, 0.20 rewatch, 0.10 likes, 0.10 comments, 0.05 shares.
- State files: `workspace/policy/policy.json`, `workspace/performance/<videoId>.json`, `./elite_templates/elite_template_vN.json` (env `ELITE_TEMPLATES_DIR` override).
- Existing behavior preserved when AVSS finds nothing to vary: cold policy + no templates + all-pinned ⇒ variant A equals today's plan.

### Task 2: Audience simulator (`src/avss/simulator.ts`)

**Files:** Create `src/avss/simulator.ts`; Test `tests/avss/simulator.test.ts`

**Interfaces:**
- Consumes: `EditPlan`, `SourceSignals`, `clamp01` (Task 1).
- Produces (used by Tasks 6, 8, 10):

```ts
export const TICK = 0.5;
export interface SimPoint { t: number; v: number; }
export interface DopamineEvent { t: number; kind: 'impact'|'reward'|'humor'|'surprise'; strength: number; }
export interface SimResult {
  attention: SimPoint[]; dopamine: DopamineEvent[]; hazard: SimPoint[]; retention: SimPoint[];
  avgRetention: number; completion: number; rewatch: number;
  rewatchFactors: { surpriseHumor: number; loopPull: number; tightness: number; endSpike: number };
  dropoffs: number[];        // top-3 hazard local-max times
  overall: number;
}
export function visualEvents(plan: EditPlan): number[];   // sorted: 0 (hook/cut-in), zoom times, broll starts+ends, emphasized word starts
export function simulate(plan: EditPlan, signals: SourceSignals): SimResult;
```

Model (all curves at `t = 0, 0.5, … ≤ durationSec`):
- helpers: `rmsAt(t)` nearest-point lookup /10 → 0–1; `wordsPerSecAt(t)` = words starting in `[t−1, t+1]` / 2, capped /2 → 0–1; `inSilence(t)`; `lastEventGap(t)` vs `visualEvents` (which always contains 0).
- **attention** = `clamp01(0.35 + 0.3·wps + 0.35·rms + boost − stale)`; `boost = 0.25·exp(−gapToPrevEvent/0.8)`; `stale = min(0.4, 0.08·max(0, gap − 2.5))`.
- **dopamine**: impact = emphasized words with `rmsAt ≥ 0.7` (strength = rms); humor = at loudest emphasized word if `semantic.humor ≥ 0.5` (strength = humor); surprise = at first emphasized word after 1 s if `semantic.surprise ≥ 0.5`; reward = at max-RMS tick in the final 25 % (strength = rms). Merge events within 1 s (keep stronger), cap 8, sort by t.
- **hazard** per tick: `h = 0.010` + `0.030` if `inSilence(t) && gap > 1.5` + `min(0.05, 0.012·max(0, gap − 2.5))` + `0.010` if `max(emotional_intensity, humor, surprise) < 0.3`. First 3 s: `h ×= 3`, then `×0.55` if hookText, `×0.8` if an emphasized word starts < 3 s, `×0.85` if `rmsAt(t) ≥ 0.6`. Clamp `[0.001, 0.25]`.
- **retention**: survival product; `avgRetention` = mean; `completion` = last; `dropoffs` = t of top-3 strict local maxima of hazard.
- **rewatch** = clamp01 of `0.4·max(humor, surprise) + 0.3·min(1, dopamineInFinalQuarter/2) + 0.2·clamp01(1 − max(0, dur − 30)/30) + 0.1·(dopamine within 2 s of end ? 1 : 0)` with the factor breakdown returned.
- **overall** = `0.35·avgRetention + 0.20·completion + 0.20·rewatch + 0.10·emotional_intensity + 0.10·max(controversy, argument_peak) + 0.05·max(humor, surprise)`.

- [ ] **Step 1: Failing tests** — monotonicity, not exact values: (a) a 20 s plan with 4 zoom events has higher mean attention than the same plan with none; (b) signals with a 5 s silence produce higher mean hazard than without; (c) retention is non-increasing and `completion == last`; (d) hookText lowers first-3 s hazard vs no hook; (e) uniform hazard 0.01 over 60 ticks ⇒ completion ≈ 0.99^60 (±1e-6); (f) high humor+surprise raises rewatch vs zeros; (g) overall ∈ [0,1]; (h) dopamine merged events ≥1 s apart, ≤8.
